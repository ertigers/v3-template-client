# 权限集使用指南

## 📋 概述

系统已从基于角色的权限控制(RBAC)升级为权限集控制，提供更细粒度和灵活的权限管理。

## 🔧 权限配置

### 权限常量定义
所有权限在 `src/config/permissions.js` 中定义：


### 菜单配置
菜单配置使用 `permissions` 字段：

```json
{
  "path": "/station/create",
  "name": "stationCreate",
  "component": "/station/create/index",
  "meta": {
    "permissions": ["station:create"],
    "title": "创建电站"
  }

```

## 🎯 使用方法

### 1. 组合式函数 (推荐)

```vue
<template>
  <div>
    <!-- 单个权限检查 -->
    <el-button v-if="can('station:create')" @click="createStation">
      创建电站
    </el-button>
    
    <!-- 多权限检查 - 任意一个 -->
    <el-button v-if="canAny(['station:edit', 'station:delete'])" @click="editStation">
      编辑/删除
    </el-button>
    
    <!-- 多权限检查 - 全部拥有 -->
    <el-button v-if="canAll(['station:view', 'station:edit'])" @click="advancedEdit">
      高级编辑
    </el-button>
  </div>
</template>

<script setup>
import { usePermissions } from '@/composables/usePermissions';

const { can, canAny, canAll, userPermissions } = usePermissions();

const createStation = () => {
  if (can('station:create')) {
    // 创建电站逻辑
  }
};
</script>
```

### 2. 权限指令

```vue
<template>
  <div>
    <!-- 基础权限指令 -->
    <el-button v-permission="'station:create'">创建电站</el-button>
    
    <!-- 多权限 - 任意一个 -->
    <el-button v-permission="['station:edit', 'station:delete']">
      编辑/删除
    </el-button>
    
    <!-- 多权限 - 全部拥有 -->
    <el-button v-permission.all="['station:view', 'station:edit']">
      高级编辑
    </el-button>
    
    <!-- 兼容旧版本auth指令 -->
    <el-button v-auth="'add'">添加</el-button>
  </div>
</template>
```

### 3. 在Pinia Store中使用

```javascript
import { useAuthStore } from '@/stores/modules/auth';

export const useStationStore = defineStore('station', () => {
  const authStore = useAuthStore();
  
  const canCreateStation = computed(() => {
    return authStore.userPermissionsGet.includes('station:create');
  });
  
  return {
    canCreateStation
  };
});
```

### 4. 在路由守卫中使用

```javascript
import { useAuthStore } from '@/stores/modules/auth';
import { hasPermission } from '@/utils';

router.beforeEach((to, from, next) => {
  const authStore = useAuthStore();
  const userPermissions = authStore.userPermissionsGet;
  
  if (hasPermission(userPermissions, to)) {
    next();
  } else {
    next('/403');
  }
});
```

## 🔄 角色兼容

系统保持与原有角色系统的兼容性：

```javascript
// 角色权限映射
export const ROLE_PERMISSION_MAP = {
  'admin': Object.values(ALL_PERMISSIONS),
  'org-user': [
    STATION_PERMISSIONS.VIEW,
    STATION_PERMISSIONS.CREATE,
    // ...
  ]
};
```

用户信息可以包含：
- `permissions`: 直接的权限集数组
- `roles`: 角色数组（会自动映射为权限集）

## ⚡ 性能优化

### 权限缓存
权限信息会自动缓存在Pinia Store中，避免重复计算。

### 按需加载
权限检查是响应式的，当权限变更时会自动更新UI。

## 🚨 注意事项

1. **权限粒度**: 不要过度细分权限，保持合理的粒度
2. **命名规范**: 使用 `模块:操作` 的命名格式，如 `station:create`
3. **兼容性**: 新老权限系统可以并存，逐步迁移
4. **安全性**: 前端权限仅用于UI控制，后端必须进行权限验证

## 📝 迁移指南

### 从角色控制迁移到权限集

1. **保持现有功能**：原有的 `v-auth` 指令和 `useAuthButtons` 仍然可用
2. **逐步迁移**：新功能使用权限集，旧功能保持不变
3. **测试验证**：确保权限控制正确生效

### 建议的迁移步骤

```javascript
// 步骤1: 在组件中同时支持两种方式
<el-button v-auth="'add'" v-permission="'station:create'">
  创建电站
</el-button>

// 步骤2: 逐步替换为权限集
<el-button v-permission="'station:create'">
  创建电站
</el-button>

// 步骤3: 使用组合式函数（最终形态）
<el-button v-if="can('station:create')">
  创建电站
</el-button>
```

## 🎉 最佳实践

1. **优先使用组合式函数**: 更灵活，更易测试
2. **权限常量化**: 使用常量避免硬编码
3. **逻辑权限检查**: 在业务逻辑中也要进行权限检查
4. **用户体验**: 没有权限的功能应该隐藏而不是禁用
